## 1. Purpose and Scope

This document defines the exact JSON payload formats required for all Gateway v59 actions to succeed on the first execution attempt. Every field requirement, constraint, and behavior described here was extracted directly from live n8n workflow code.

**Covered actions:** `artifact.save`, `artifact.query`, `artifact.list`, `artifact.update`, `artifact.promote`, `artifact.delete`, `artifact.restore`, `artifact.list_deleted`

**Not covered:** Workflow implementation details (n8n nodes), RLS policy enforcement, database trigger internals, error recovery procedures.

## 2. Global Payload Rules

### 2.1 Always Required Fields (All Actions)

| Field | Type | Description |
|-------|------|-------------|
| `gw_action` | text | Action identifier. One of the 8 supported actions. |
| `gw_workspace_id` | uuid | Target workspace. Required for all actions. |
| `artifact_type` | text | Must be from the Gateway TYPE_ALLOWLIST (see 2.3). |

### 2.2 Allowed Actions (Gateway ACTION_ALLOWLIST)

```
artifact.save
artifact.query
artifact.list
artifact.update
artifact.promote
artifact.delete
artifact.restore
artifact.list_deleted
```

Any other `gw_action` value returns `ACTION_NOT_ALLOWED` (HTTP 403).

### 2.3 Allowed Artifact Types (Gateway TYPE_ALLOWLIST)

**8 types permitted by the Gateway Gatekeeper:**

```
project
journal
restart
snapshot
instruction_pack
branch
limb
leaf
```

**5 types in DDL CHECK but NOT in Gateway allowlist:** `grass`, `thorn`, `forest`, `thicket`, `flower`. These return `ARTIFACT_TYPE_NOT_ALLOWED` at the Gateway.

### 2.4 Forbidden Fields (Server-Owned)

| Field | Reason |
|-------|--------|
| `artifact_id` on save INSERT | Generated by database via `gen_random_uuid()` |
| `owner_user_id` | Forced to MVP service principal on INSERT |
| `version` | System-managed (DB default + workflow increment) |
| `created_at` | Database default `now()` |
| `updated_at` | Database trigger on every UPDATE |
| `workspace_id` | Derived from `gw_workspace_id` |

### 2.5 Sequential Action Discipline

When performing multiple dependent actions (e.g., project save then snapshot save with parent_artifact_id):

1. Execute the first action
2. Wait for Gateway confirmation
3. Extract the returned `artifact_id`
4. Use that exact ID in subsequent payloads

**Never** invent UUIDs, assume save success, or continue without confirmation.

### 2.6 Chrome Extension Raw JSON Rule

Payloads for Chrome Extension execution must be:
- Raw JSON only (starts with `{`, ends with `}`)
- No markdown fences, headings, labels, or comments
- One payload per execution
- Sequential discipline enforced between dependent payloads

### 2.7 Field Aliases (Accepted but Not Recommended)

The Gateway Normalize_Request accepts these aliases:

| Alias | Canonical Field |
|-------|----------------|
| `action` | `gw_action` |
| `workspace_id` | `gw_workspace_id` |
| `patch` | `extension` |

Always prefer canonical field names.

## 3. artifact.save

**Purpose:** Create a new artifact (INSERT into spine + extension table).

**Note:** Save also has an internal UPDATE path triggered when `artifact_id` is present. The canonical contract is CREATE — use `artifact.update` for modifications.

### 3.1 Required Fields

| Field | Type | Notes |
|-------|------|-------|
| `gw_action` | `"artifact.save"` | |
| `gw_workspace_id` | uuid | |
| `artifact_type` | text | Must be in TYPE_ALLOWLIST |
| `title` | text | Non-empty string |
| `semantic_type_id` | uuid | **REQUIRED for top-level types** (project, snapshot, journal, restart). **FORBIDDEN for non-top-level types** (branch, leaf, limb, instruction_pack, forest, thicket, flower). See 3.2.1. |
| `extension` | object | Type-specific. See 3.2 for requirements per type. |

### 3.2 Extension Requirements by Artifact Type

| Type | Required Extension Fields | Notes |
|------|--------------------------|-------|
| `project` | `lifecycle_stage` (string: `seed`, `sapling`, `tree`, `archive`) | Normalizer aligns `lifecycle_status` from this value |
| `journal` | `entry_text` (non-empty string) | **Strict allow-list: ONLY `entry_text` permitted.** Any other key triggers `JOURNAL_EXTENSION_INVALID`. |
| `snapshot` | `payload` (non-null, non-array object) | Extension must contain a `payload` object with the snapshot content |
| `restart` | `payload` (non-null, non-array object) | Extension must contain a `payload` object with the restart content |
| `instruction_pack` | `scope` (string), `active` (boolean), `priority` (number), `pack_format` (string) | `pack_format` is validated but hardcoded to `"json"` on write |
| `branch` | *(none — spine-only)* | No extension table write. Extension object is ignored. |
| `leaf` | *(none — spine-only)* | No extension table write. Extension object is ignored. |
| `limb` | *(none required)* | Shell INSERT: only `artifact_id` written to extension table |

### 3.2.1 Semantic Type Classification (T69)

**Top-level types** (REQUIRE `semantic_type_id` on INSERT):
- `project`, `snapshot`, `journal`, `restart`

**Non-top-level types** (MUST NOT provide `semantic_type_id`):
- `branch`, `leaf`, `limb`, `instruction_pack`, `forest`, `thicket`, `flower`

The `semantic_type_id` must be a valid UUID referencing an **active** entry in `qxb_semantic_type_registry`. If the registry entry does not exist or is inactive, the save is rejected.

**Current registry values (9 active):**
- `execution-core` — Operational execution and task management
- `governance` — Rules, policies, structural decisions
- `infrastructure` — Technical systems and tooling
- `platform` — Platform capabilities and integration
- `product` — Product features and user-facing functionality
- `alignment` — Strategic alignment and prioritization
- `sales` — Sales strategy and pipeline
- `marketing` — Marketing strategy and outreach
- `exploratory` — Exploration, experimentation, discovery

### 3.3 Optional Spine Fields

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `summary` | text | `null` | |
| `priority` | integer 1-5 | `3` | Workflow defaults to 3 if omitted. DB CHECK enforces range. |
| `tags` | array of strings | `null` | Normalized: trimmed, lowercased, deduplicated |
| `content` | object | `{}` | Must be an object (not string/array) |
| `parent_artifact_id` | uuid | `null` | Links to parent artifact |
| `lifecycle_status` | text | `null` | Auto-aligned from `extension.lifecycle_stage` for project |
| `execution_status` | text | `null` | Must be one of: `not_started`, `in_progress`, `blocked`, `complete` (or null) |

### 3.4 Server-Generated Fields

| Field | Generated By |
|-------|-------------|
| `artifact_id` | Database `gen_random_uuid()` |
| `version` | Database default (1) |
| `created_at` | Database `now()` |
| `updated_at` | Database trigger |
| `owner_user_id` | Forced to MVP service principal |

### 3.5 Version Behavior

Save INSERT: version = database default (1). The workflow does not write version.

### 3.6 Success Response

```json
{
  "ok": true,
  "gw_action": "artifact.save",
  "artifact_id": "<uuid>",
  "artifact_type": "<type>",
  "workspace_id": "<uuid>",
  "operation": "INSERT",
  "timestamp": "<ISO 8601>"
}
```

For snapshot/restart, the response also includes `"extension": { "payload": {...} }`.

## 4. artifact.query

**Purpose:** Retrieve a single artifact by ID with optional extension hydration.

### 4.1 Required Fields

| Field | Type | Notes |
|-------|------|-------|
| `gw_action` | `"artifact.query"` | |
| `gw_workspace_id` | uuid | |
| `artifact_type` | text | Must match the stored artifact's type |
| `artifact_id` | uuid | |

### 4.2 Selector

| Field | Type | Default | Behavior |
|-------|------|---------|----------|
| `selector.hydrate` | boolean | `true` (implicit) | `false` (strict boolean) skips extension fetch, returns `extension: null`. Any other falsy value is treated as truthy. |

No other selector fields are consumed by the Query sub-workflow.

### 4.3 Type Mismatch Handling

The workflow compares `requested_artifact_type` against the `stored_artifact_type` from the spine. If they don't match, `TYPE_MISMATCH` error is returned.

**Smart fallback:** If a type has no extension table routing (e.g., grass, thorn) but the stored and requested types match, the query succeeds with `extension: null` (spine-only).

### 4.4 Success Response

Returned inside a Gateway wrapper: `{ "ok": true, "_gw_route": "ok", "data": { "artifact": { ...fields... } } }`

The artifact object contains all spine fields plus:
- `extension: { ...type_fields... }` when hydrated
- `extension: null` when not hydrated or spine-only type
- `semantic_type_id` on the spine (returned for all artifact types; null for non-top-level)

### 4.5 Minimal Valid Payloads

**Query with hydration (default):**
```json
{
  "gw_action": "artifact.query",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "artifact_id": "<uuid>"
}
```

**Query without hydration:**
```json
{
  "gw_action": "artifact.query",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "snapshot",
  "artifact_id": "<uuid>",
  "selector": {
    "hydrate": false
  }
}
```

**Would-fail example — wrong artifact_type:**
```json
{
  "gw_action": "artifact.query",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "journal",
  "artifact_id": "<uuid_of_a_project>"
}
```
*Fails with:* `TYPE_MISMATCH` — requested type does not match stored type.

## 5. artifact.list

**Purpose:** List artifacts by type with optional filtering, pagination, and hydration.

### 5.1 Required Fields

| Field | Type | Notes |
|-------|------|-------|
| `gw_action` | `"artifact.list"` | |
| `gw_workspace_id` | uuid | |
| `artifact_type` | text | Can be in `selector.artifact_type` or top-level |

### 5.2 Selector

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `selector.limit` | integer | `50` | Max: 200. Clamped silently if exceeded. Must be >= 1. |
| `selector.offset` | integer | `0` | Must be >= 0. |
| `selector.hydrate` | boolean | `false` | `true` (strict boolean) fetches extension data. |
| `selector.as_of` | ISO datetime | auto-generated | Paging anchor. Pass back from response for consistent pagination. |
| `selector.include_fields` | array | `null` | Field projection — only listed fields returned per artifact. |
| `selector.filters.tags_any` | array of strings | `null` | **Set containment filter (all-of, not any-of).** Artifact must contain ALL specified tags. |
| `selector.parent_artifact_id` | uuid | `null` | Filter by parent artifact. |

**Pagination cap:** `offset + limit + 1` must not exceed 500 (`FETCH_CAP`). Violation returns `PAGINATION_WINDOW_EXCEEDED`.

### 5.3 Tag Filter Semantics

Despite the field name `tags_any`, the PostgREST `cs.` operator performs **set containment** (tags_all). An artifact must contain ALL specified tags to match, not just any one of them.

### 5.4 Hydration by Type

| Type | Hydration Supported | Extension Table |
|------|-------------------|-----------------|
| `project` | Yes | `qxb_artifact_project` |
| `journal` | Yes | `qxb_artifact_journal` |
| `restart` | Yes | `qxb_artifact_restart` |
| `snapshot` | Yes | `qxb_artifact_snapshot` |
| `instruction_pack` | Yes | `qxb_artifact_instruction_pack` |
| `limb` | Yes | `qxb_artifact_limb` |
| `branch` | Spine-only | `extension: null` |
| `leaf` | Spine-only | `extension: null` |

### 5.5 Success Response

```json
{
  "ok": true,
  "gw_action": "artifact.list",
  "gw_workspace_id": "<uuid>",
  "artifact_type": "<type>",
  "selector": { "limit": 50, "offset": 0, "hydrate": false, "as_of": "<ISO>" },
  "data": {
    "artifacts": [ ... ]
  },
  "meta": {
    "count": 10,
    "limit": 50,
    "offset": 0,
    "has_more": false,
    "as_of": "<ISO>"
  },
  "timestamp": "<ISO>"
}
```

`meta.count` is page count (not total). `meta.has_more` indicates if more pages exist.

**Basic list:**
```json

  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",

    "limit": 20

```

## 6. artifact.update

**Purpose:** Modify an existing artifact's spine fields, tags, extension fields, or semantic_type_id. Some modes can be combined (spine + tags); others require separate calls.

### 6.1 Required Fields

| Field | Type | Notes |
|-------|------|-------|
| `gw_action` | `"artifact.update"` | |
| `gw_workspace_id` | uuid | |
| `artifact_type` | text | |
| `artifact_id` | uuid | |
| `extension` OR `tags` OR spine fields | object | At least one mutation target required. See 6.2 for mode selection. |

### 6.2 Mode Selection

The workflow determines update mode automatically based on which fields are present:

| Mode | Trigger | What It Updates |
|------|---------|----------------|
| `tags_only` | `tags` present, no `extension`, no spine fields | Tags only |
| `spine_only` | Spine field(s) present (`title`, `summary`, `priority`), no `extension`, no `tags` | Spine fields only |
| `mixed` | Spine field(s) + `tags` both present, no `extension` | Spine fields + tags atomically |
| `extension` | `extension` has key(s) other than `semantic_type_id` | Extension fields only |
| `semantic_type` | `extension` contains `semantic_type_id` key | Semantic type only (dedicated path, see 6.5) |
| `extension` (with tags) | `extension` + `tags` both present | Extension + tags (two separate DB writes) |

**Combination rules:**
- Spine fields + tags = allowed (atomic `mixed` mode)
- Spine fields + extension = **NOT allowed** (`MIXED_UPDATE_NOT_ALLOWED`)
- Extension + tags = allowed (sequential writes)
- Semantic type + anything else = **NOT allowed** (`MIXED_UPDATE_NOT_ALLOWED`)

### 6.3 Tags-Only Update

Works for **ALL artifact types** including immutable ones (snapshot, restart, journal).

**Shape:**
```json
{
  "tags": {
    "add": ["tag1", "tag2"],
    "remove": ["old-tag"]
  }
}
```

**Semantics:**
- Tags in `add` are appended (deduplicated)
- Tags in `remove` are deleted
- **Remove wins over add** if same tag appears in both
- Final tags sorted alphabetically
- Each tag is trimmed and lowercased
- Empty array is valid (clears all tags if all removed)

**Version:** Incremented by 1 on every tags-only update.

**Lifecycle exception:** Tags-only updates are blocked on `archive` artifacts (`ARCHIVE_IMMUTABLE`). All other lifecycle stages permit tags-only updates, including on immutable extension types (snapshot, restart, journal).

### 6.4 Extension Update

**Only `project` has real extension persistence.** All other types either error or return ack-without-write.

#### 6.4.1 Mutability Registry

| Type | Extension Update Behavior |
|------|--------------------------|
| `project` | Allowed. Only `operational_state`, `state_reason`, and `design_spine` fields. |
| `snapshot` | **BLOCKED** — `IMMUTABILITY_ERROR` (CREATE_ONLY) |
| `restart` | **BLOCKED** — `IMMUTABILITY_ERROR` (CREATE_ONLY) |
| `journal` | **BLOCKED** — `JOURNAL_INSERT_ONLY` (permanent INSERT-only doctrine) |
| `branch` | Returns `UPDATE_CONFIRMED` ack but **NO database write, NO version increment** |
| `limb` | Returns `UPDATE_CONFIRMED` ack but **NO database write, NO version increment** |
| `leaf` | Returns `UPDATE_CONFIRMED` ack but **NO database write, NO version increment** |
| `instruction_pack` | **BLOCKED** — `IMMUTABILITY_ERROR` (CREATE_ONLY) |
| `grass`, `thorn`, others | **Silent dead-end** — no response produced |

#### 6.4.2 Project Extension — Allowed Fields

| Field | Behavior |
|-------|----------|
| `operational_state` | Written to `qxb_artifact_project.operational_state` |
| `state_reason` | Written to `qxb_artifact_project.state_reason` |
| `design_spine` | Written to `qxb_artifact_project.design_spine` (JSONB) |
| `lifecycle_stage` | **BLOCKED** — `MUTABILITY_ERROR` with `PROMOTE_ONLY` hint |
| Any other field | **BLOCKED** — `MUTABILITY_ERROR` with disallowed field list |

**Version:** Incremented by 1 on successful project extension update.

### 6.5 Semantic Type Update (Dedicated Path — T69)

Changes the `semantic_type_id` of an existing top-level artifact. This is a **dedicated, isolated update path** — it cannot be combined with tags or other extension fields.

**Applies to:** `project`, `snapshot`, `journal`, `restart` only. Non-top-level types return `SEMANTIC_TYPE_NOT_APPLICABLE`.

**Required fields:**
```json
{
  "gw_action": "artifact.update",
  "gw_workspace_id": "<uuid>",
  "artifact_type": "<top-level type>",
  "artifact_id": "<uuid>",
  "extension": {
    "semantic_type_id": "<new_semantic_type_id>",
    "reason": "<non-empty string>"
  }
}
```

**Rules:**
- `reason` is REQUIRED (non-empty string explaining the reclassification)
- `semantic_type_id` and `reason` must be the ONLY keys in `extension` — any other key returns `MIXED_UPDATE_NOT_ALLOWED`
- `tags` must NOT be present — combining tags with semantic type returns `MIXED_UPDATE_NOT_ALLOWED`
- The new `semantic_type_id` must be a valid, active entry in `qxb_semantic_type_registry`
- If the new value equals the current value, the operation is a **noop** (no version increment, no audit entry)

**Version:** Incremented by 1 via `update_semantic_type()` RPC on true change only.

**Audit:** On true change, an entry is inserted into `qxb_semantic_type_audit` with old/new values and reason.

**Success response (true change):**
```json
{
  "ok": true,
  "gw_action": "artifact.update",
  "artifact_id": "<uuid>",
  "artifact_type": "<type>",
  "operation": "SEMANTIC_TYPE_UPDATE",
  "old_semantic_type_id": "<uuid>",
  "new_semantic_type_id": "<uuid>",
  "version": 3,
  "timestamp": "<ISO>"
}
```

**Success response (noop):**
```json
{
  "ok": true,
  "gw_action": "artifact.update",
  "artifact_id": "<uuid>",
  "artifact_type": "<type>",
  "operation": "SEMANTIC_TYPE_UPDATE",
  "noop": true,
  "message": "semantic_type_id unchanged",
  "timestamp": "<ISO>"
}
```

### 6.6 Spine Field Updates (T87)

The following spine fields are now updateable via `artifact.update` in `spine_only` or `mixed` mode:

| Field | Type | Notes |
|-------|------|-------|
| `title` | text | Non-empty string. **Frozen at `tree` lifecycle** (`FIELD_FROZEN`). |
| `summary` | text | Nullable. |
| `priority` | integer 1-5 | DB CHECK enforces range. |

**NOT updateable via artifact.update:** `execution_status`, `parent_artifact_id`, `lifecycle_status`, `content`. These remain accessible only via `artifact.save` UPDATE path (not recommended) or dedicated actions (promote).

**Lifecycle restrictions (see 6.8):**
- `archive`: ALL spine field updates blocked (`ARCHIVE_IMMUTABLE`)
- `tree`: `title` frozen; `summary` and `priority` remain mutable
- `seed`/`sapling`: all three fields fully mutable

**Version:** Incremented by 1 on successful spine field update.

### 6.8 Lifecycle-Scoped Mutability (T87)

Mutation permissions vary by the artifact's current `lifecycle_status`:

| Lifecycle | Spine Fields | Extension Fields | Tags | Semantic Type |
|-----------|-------------|-----------------|------|---------------|
| `seed` | Fully mutable | Per type registry | Mutable | Mutable |
| `sapling` | Fully mutable | Per type registry | Mutable | Mutable |
| `tree` | `title` FROZEN; `summary`, `priority` mutable | Per type registry | Mutable | Mutable |
| `archive` | **ALL BLOCKED** | **ALL BLOCKED** | **ALL BLOCKED** | **ALL BLOCKED** |

**Enforcement:**
- `archive` check runs FIRST, before any mode-specific logic. Any mutation attempt returns `ARCHIVE_IMMUTABLE`.
- `tree` + `title` check runs during spine field validation. Returns `FIELD_FROZEN` with details indicating which field and why.
- These checks apply to ALL artifact types that have lifecycle_status (primarily projects, but the guard is generic).

**QPM interaction:** Promotion guards (Section 7.3) are unaffected. Lifecycle-scoped mutability governs updates, not promotions.

## 7. artifact.promote

**Purpose:** Advance a project's lifecycle stage (seed -> sapling -> tree -> archive).

### 7.1 Required Fields

| Field | Type | Notes |
|-------|------|-------|
| `gw_action` | `"artifact.promote"` | |
| `gw_workspace_id` | uuid | |
| `artifact_type` | text | Typically `"project"` |
| `artifact_id` | uuid | |
| `transition` | text | One of the valid transition keys (see 7.2). Accepted at top level. |
| `reason` | text | Non-empty, 1-280 characters. Accepted at top level. |

**Note:** The Gateway normalizer bridges top-level `transition`/`reason` into `artifact_payload.transition`/`artifact_payload.reason` for the Promote sub-workflow. Clients should use top-level fields.

**Optional:** `actor_user_id` (uuid) — if provided, must be a valid UUID. Written to the event log.

### 7.2 Valid Transitions

| Transition | From | To |
|-----------|------|----|
| `seed_to_sapling` | `seed` | `sapling` |
| `sapling_to_tree` | `sapling` | `tree` |
| `tree_to_archive` | `tree` | `archive` |

Only forward transitions are allowed. Archive is terminal and irreversible.

### 7.3 QPM Guards (Qwrk Promotion Model)

| Transition | Guard | Requirement |
|-----------|-------|-------------|
| `seed_to_sapling` | Summary OR journal child | Artifact must have a non-empty `summary` field on the spine, OR at least 1 journal-type child artifact (linked via `parent_artifact_id`). |
| `sapling_to_tree` | Execution anatomy children | At least 1 child artifact of type `branch` or `leaf` (linked via `parent_artifact_id`). **`limb` is NOT counted.** |
| `tree_to_archive` | *(none)* | No guards. Passes unconditionally. |

### 7.4 State Verification

Before promotion, the workflow verifies:
1. The artifact exists (by `artifact_id` + `workspace_id`)
2. The current `lifecycle_status` matches the expected `from_state` for the requested transition
3. If mismatch: `LIFECYCLE_STATE_MISMATCH` error

### 7.5 Version Behavior

Promote increments version by 1.

### 7.6 Event Logging

Every successful promotion creates an event in `qxb_artifact_event`:
- `event_type`: `"lifecycle_promote"`
- `payload`: `{ transition, from_state, to_state, reason, artifact_type, gw_action }`
- `actor_user_id`: from request (or null)

### 7.7 Hydration

The `selector.hydrate` field is accepted but **not implemented** — promote does not re-fetch the artifact after update. The response is shaped from the event insert result.

### 7.8 Success Response

```json
{
  "ok": true,
  "gw_action": "artifact.promote",
  "workspace_id": "<uuid>",
  "artifact_type": "project",
  "artifact_id": "<uuid>",
  "event_id": "<uuid>",
  "operation": "PROMOTE",
  "from_state": "seed",
  "to_state": "sapling",
  "transition": "seed_to_sapling",
  "timestamp": "<ISO>",
  "data": {
    "event": { ... }
  }
}
```

### 7.9 Minimal Valid Payloads

**Promote seed to sapling:**
```json
{
  "gw_action": "artifact.promote",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
```

## 11. Tag Semantics

### 11.1 Tag Normalization (On Save)

- Tags are trimmed and lowercased
- Comma-separated strings within array elements are split
- Duplicates removed via Set
- Empty strings filtered out

### 11.2 Tag Update Semantics

- `tags.add`: array of strings to append (deduplicated with existing)
- `tags.remove`: array of strings to delete
- Remove wins over add (remove runs after add)
- Final array sorted alphabetically
- Both arrays normalized: strings only, trimmed, deduplicated

### 11.3 Tag Querying

- `artifact.list` with `selector.filters.tags_any`: Set containment (all-of)
- No tag-based querying in `artifact.query` (single-artifact fetch)
