# Qwrk Gateway JSON Payload Canonical Reference v1

**Date:** 2026-01-17
**Status:** Authoritative
**Kernel:** Gateway v1 (KGB-Locked)

---

## Contract Alignment

This document reflects Gateway v1 canonical payload behavior. It supersedes earlier draft contracts where ID generation or update semantics differed. Specifically: `artifact.save` is CREATE ONLY; updates are performed exclusively via `artifact.update`; and `artifact_id` is generated by the database on insert, not by the Gateway or client.

---

## 1. Purpose and Scope

### Why This Document Exists

This document defines the exact JSON payload formats required for all Gateway v1 actions to succeed on the first execution attempt. It consolidates truth from multiple authoritative sources into a single reference.

### What Problems It Prevents

- Payload guessing and trial-and-error debugging
- Field name mismatches between client and server expectations
- Incorrect optionality assumptions
- Mutation attempts on immutable or restricted fields
- Type mismatch errors from incorrect artifact_type handling

### What It Explicitly Does NOT Cover

- Workflow implementation details (n8n node configurations)
- RLS policy enforcement mechanisms
- Database trigger behavior
- Error recovery procedures
- Future actions (artifact.delete, artifact.archive)

## 2. Global Payload Rules (Applies to ALL Actions)

### 2.1 Always Required Fields

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| `gw_action` | text | Client | Action identifier (save, query, update, list, promote) |
| `gw_workspace_id` | uuid | Client | Target workspace for the request |
| `artifact_type` | text | Client | Must be from allowed artifact types |

### 2.2 Conditionally Required Fields

| Field | Condition | Type | Description |
|-------|-----------|------|-------------|
| `artifact_id` | Required for query, update, promote; Forbidden for save | uuid | Target artifact identifier |
| `extension` | Required for save, update | object | Type-specific payload |
| `title` | Required for save on most types | text | Human-readable title |
| `transition` | Required for promote | text | Lifecycle transition key |

### 2.3 Forbidden from Clients (Server-Owned)

| Field | Reason | Source |
|-------|--------|--------|
| `artifact_id` on save | Generated by database; save is CREATE ONLY | Phase 3 Gateway Contract P3-D2 |
| `gw_user_id` | Derived from auth context; client must not supply | Gateway v1 Auth Model |
| `version` | System-managed | North Star, Mutability Registry |
| `created_at` | System-managed | Mutability Registry SYSTEM_ONLY |
| `updated_at` | System-managed | Mutability Registry SYSTEM_ONLY |
| `owner_user_id` | Set from auth context | Mutability Registry SYSTEM_ONLY |
| `workspace_id` | Derived from gw_workspace_id | Mutability Registry SYSTEM_ONLY |

**Note on actor identity:** The resolved actor user is derived server-side from auth context. Client-supplied user identifiers (including `gw_user_id`) are not accepted in request payloads and will be ignored or rejected.

### 2.4 Server-Owned vs Client-Supplied Truth

| Truth Owner | Fields | Notes |
|-------------|--------|-------|
| Server | artifact_id, version, timestamps, owner_user_id | Never accept from client |
| Client | title, summary, tags, content, extension fields | Validated then persisted |
| Derived | workspace_id (from gw_workspace_id) | Client sends gw_workspace_id |

### 2.5 ID Generation Rules

- `artifact_id`: Generated by PostgreSQL database default (`gen_random_uuid()`) on INSERT; Gateway does not generate IDs
- `gw_request_id`: Optional client-supplied correlation ID (echoed in response)
- Client must never supply `artifact_id` for `artifact.save`; the field is forbidden on create
- For `artifact.save`, the response includes the database-generated `artifact_id`

### 2.6 Parent / Lineage Rules

| Parent Type | Valid Child Types | Enforcement |
|-------------|-------------------|-------------|
| forest | thicket | Database constraint |
| thicket | project, flower | Database constraint |
| project | snapshot, restart | Semantic (Gateway) |
| (none) | journal | Journals have no parent requirement |

## 3. Gateway Action Payload Specs

---

### 3.1 artifact.save (CREATE ONLY)

`artifact.save` is exclusively for creating new artifacts. It does not support updates. To update an existing artifact, use `artifact.update`.

#### 3.1.1 Canonical JSON Shape

```json
{
  "gw_action": "artifact.save",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "title": "My Project Title",
  "summary": "Brief description of the project",
  "tags": ["tag1", "tag2"],
  "content": {},
  "parent_artifact_id": null,
  "priority": 3,
  "lifecycle_status": "seed",
  "extension": {
    "lifecycle_stage": "seed",
    "operational_state": "active",
    "state_reason": null
  }
}
```

**Note:** `artifact_id` must NOT be included. The database generates and returns the ID in the response.

#### 3.1.2 Field-by-Field Semantics

| Field | Required | Type | Source | Governing Document |
|-------|----------|------|--------|-------------------|
| `gw_action` | Yes | text | Client | Gateway Contract v1 |
| `gw_workspace_id` | Yes | uuid | Client | Gateway Contract v1 |
| `artifact_type` | Yes | text | Client | Gateway Contract v1, LIVE_DDL constraint |
| `artifact_id` | Forbidden | uuid | N/A | Save is CREATE ONLY; ID generated by DB |
| `title` | Yes (create) | text | Client | LIVE_DDL: qxb_artifact.title NOT NULL |
| `summary` | No | text | Client | LIVE_DDL: qxb_artifact.summary nullable |
| `tags` | No | jsonb array | Client | LIVE_DDL: qxb_artifact.tags nullable |
| `content` | No | jsonb | Client | LIVE_DDL: qxb_artifact.content nullable |
| `parent_artifact_id` | Depends on type | uuid | Client | North Star lineage rules |
| `priority` | No | int (1-5) | Client | LIVE_DDL: CHECK 1-5, defaults to 3 |
| `lifecycle_status` | No | text | Client | LIVE_DDL: qxb_artifact.lifecycle_status |
| `extension` | Yes (create) | object | Client | Type-specific, see section 3.1.3 |

#### 3.1.3 Artifact-Type Variations

**project:**
```json
{
  "extension": {
    "lifecycle_stage": "seed",
    "operational_state": "active",
    "state_reason": null
  }
}
```

| Extension Field | Required | Values | Governing Document |
|-----------------|----------|--------|-------------------|
| `lifecycle_stage` | Yes | seed, sapling, tree, retired | LIVE_DDL CHECK constraint |
| `operational_state` | No | active, paused, blocked, waiting | LIVE_DDL CHECK constraint, defaults to active |
| `state_reason` | No | text | Phase 2 Type Schemas D2.2 |

**journal:**
```json
{
  "extension": {
    "entry_text": "Journal content here",
    "payload": {}
  }
}
```

| Extension Field | Required | Type | Notes |
|-----------------|----------|------|-------|
| `entry_text` | No | text | Main journal content |
| `payload` | No | jsonb | Additional structured data |

**snapshot:**
```json
{
  "extension": {
    "payload": {
      "frozen_project_state": {}
    }
  }
}
```

| Extension Field | Required | Type | Notes |
|-----------------|----------|------|-------|
| `payload` | Yes | jsonb | Frozen payload, immutable after creation |

**restart:**
```json
{
  "extension": {
    "payload": {
      "frozen_state": {},
      "restart_reason": "Session handoff",
      "next_step": "Continue from here"
    }
  }
}
```

| Extension Field | Required | Type | Notes |
|-----------------|----------|------|-------|
| `payload` | Yes | jsonb | Frozen payload, immutable after creation |

#### 3.1.4 Forbidden / Common Failure Patterns

| Pattern | Why It Fails | Correct Approach |
|---------|--------------|------------------|
| Supplying `artifact_id` | Save is CREATE ONLY; ID is forbidden | Omit artifact_id entirely; use artifact.update for existing records |
| Attempting update via save | Save does not support updates | Use artifact.update for modifications |
| Supplying `gw_user_id` | User derived from auth context | Omit; server resolves actor identity |
| Supplying `version` | Server-managed | Never include version |
| `lifecycle_stage` without `lifecycle_status` | Both should align | Set both or let server default |
| Empty `extension` for project | lifecycle_stage required | Always include lifecycle_stage |

### 3.2 artifact.query

#### 3.2.1 Canonical JSON Shape

```json
{
  "gw_action": "artifact.query",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "artifact_id": "668bd18f-4424-41e6-b2f9-393ecd2ec534"
}
```

**With hydration selector:**
```json
{
  "gw_action": "artifact.query",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "artifact_id": "668bd18f-4424-41e6-b2f9-393ecd2ec534",
  "selector": {
    "hydrate": true
  }
}
```

#### 3.2.2 Field-by-Field Semantics

| Field | Required | Type | Source | Notes |
|-------|----------|------|--------|-------|
| `gw_action` | Yes | text | Client | Must be "artifact.query" |
| `gw_workspace_id` | Yes | uuid | Client | Target workspace |
| `artifact_type` | Yes | text | Client | Used for type validation |
| `artifact_id` | Yes | uuid | Client | Target artifact |
| `selector` | No | object | Client | Optional query modifiers |
| `selector.hydrate` | No | boolean | Client | Return full hydrated payload |
| `selector.base_only` | No | boolean | Client | Return spine fields only |

#### 3.2.3 Artifact-Type Variations

All types use the same request structure. Response varies by type (extension fields differ).

#### 3.2.4 Forbidden / Common Failure Patterns

| Pattern | Why It Fails | Error Code |
|---------|--------------|------------|
| Wrong artifact_type for ID | Type validation fails | TYPE_MISMATCH |
| Invalid artifact_id | Not found | NOT_FOUND |
| Journal query by non-owner | RLS blocks | NOT_FOUND (no leak) |
| Missing artifact_id | Validation error | VALIDATION_ERROR |

### 3.3 artifact.update (ONLY Update Path)

`artifact.update` is the exclusive mechanism for modifying existing artifacts. `artifact.save` does not support updates.

#### 3.3.1 Canonical JSON Shape

```json
{
  "gw_action": "artifact.update",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "artifact_id": "e9601873-9f71-4843-bd81-9ecaccbbf9e3",
  "extension": {
    "operational_state": "paused",
    "state_reason": "Waiting for external dependency"
  }
}
```

#### 3.3.2 Field-by-Field Semantics

| Field | Required | Type | Source | Notes |
|-------|----------|------|--------|-------|
| `gw_action` | Yes | text | Client | Must be "artifact.update" |
| `gw_workspace_id` | Yes | uuid | Client | Target workspace |
| `artifact_type` | Yes | text | Client | Must be "project" (v1) |
| `artifact_id` | Yes | uuid | Client | Target artifact (must exist) |
| `extension` | Yes | object | Client | Fields to update |

**Allowed extension fields for project:**

| Field | Allowed | Governed By |
|-------|---------|-------------|
| `operational_state` | Yes | Mutability Registry UPDATE_ALLOWED |
| `state_reason` | Yes | Mutability Registry UPDATE_ALLOWED |
| `lifecycle_stage` | No | Mutability Registry PROMOTE_ONLY |

#### 3.3.3 Artifact-Type Variations

| Type | Update Allowed | Governing Document |
|------|----------------|-------------------|
| project | Yes (restricted fields) | Mutability Registry v1 |
| journal | No (UNDECIDED_BLOCKED) | Mutability Registry v1 |
| snapshot | No (CREATE_ONLY) | Mutability Registry v1, Phase 1 D2 |
| restart | No (CREATE_ONLY) | Mutability Registry v1, Phase 1 D3 |

#### 3.3.4 Forbidden / Common Failure Patterns

| Pattern | Why It Fails | Error |
|---------|--------------|-------|
| Updating lifecycle_stage | PROMOTE_ONLY | IMMUTABILITY_ERROR |
| Updating snapshot | CREATE_ONLY | IMMUTABILITY_ERROR |
| Updating restart | CREATE_ONLY | IMMUTABILITY_ERROR |
| Updating journal | UNDECIDED_BLOCKED | IMMUTABILITY_ERROR |
| Missing artifact_id | UPDATE-ONLY semantics | VALIDATION_ERROR |

### 3.4 artifact.list

#### 3.4.1 Canonical JSON Shape

**Basic list:**
```json
{
  "gw_action": "artifact.list",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project"
}
```

**With pagination:**
```json
{
  "gw_action": "artifact.list",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "selector": {
    "limit": 10,
    "offset": 0,
    "as_of": "2026-01-17T00:00:00Z"
  }
}
```

**With hydration:**
```json
{
  "gw_action": "artifact.list",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "selector": {
    "limit": 10,
    "offset": 0,
    "hydrate": true
  }
}
```

#### 3.4.2 Field-by-Field Semantics

| Field | Required | Type | Source | Notes |
|-------|----------|------|--------|-------|
| `gw_action` | Yes | text | Client | Must be "artifact.list" |
| `gw_workspace_id` | Yes | uuid | Client | Target workspace |
| `artifact_type` | Yes | text | Client | Filter by type |
| `selector` | No | object | Client | Pagination and hydration |
| `selector.limit` | No | int | Client | Max items to return |
| `selector.offset` | No | int | Client | Skip N items |
| `selector.as_of` | No | timestamptz | Client | Anchor for deterministic paging |
| `selector.hydrate` | No | boolean | Client | Include extension fields |

#### 3.4.3 Artifact-Type Variations

All types use the same request structure.

#### 3.4.4 Forbidden / Common Failure Patterns

| Pattern | Why It Fails | Correct Approach |
|---------|--------------|------------------|
| Omitting as_of on subsequent pages | Non-deterministic results | Reuse as_of from first page |
| limit > 500 | Exceeds deterministic cap | Use pagination |
| Expecting items[] in response | Wrong envelope | Use data.artifacts[] |

### 3.5 artifact.promote

#### 3.5.1 Canonical JSON Shape

```json
{
  "gw_action": "artifact.promote",
  "gw_workspace_id": "be0d3a48-c764-44f9-90c8-e846d9dbbd0a",
  "artifact_type": "project",
  "artifact_id": "7a0492cb-7fc5-4bca-b29c-17040803ddd7",
  "transition": "seed_to_sapling",
  "reason": "Ready for structured development"
}
```

#### 3.5.2 Field-by-Field Semantics

| Field | Required | Type | Source | Notes |
|-------|----------|------|--------|-------|
| `gw_action` | Yes | text | Client | Must be "artifact.promote" |
| `gw_workspace_id` | Yes | uuid | Client | Target workspace |
| `artifact_type` | Yes | text | Client | Must be "project" (v1) |
| `artifact_id` | Yes | uuid | Client | Target artifact |
| `transition` | Yes | text | Client | Transition key |
| `reason` | No | text | Client | Recorded in event payload |

**Valid transition values:**

| Transition Key | From State | To State | Governing Document |
|----------------|------------|----------|-------------------|
| `seed_to_sapling` | seed | sapling | Phase 1 D4 |
| `sapling_to_tree` | sapling | tree | Phase 1 D4 |
| `tree_to_retired` | tree | retired | Phase 1 D4 |
| `retired_to_tree` | retired | tree | Phase 1 D1 (admin-only) |

#### 3.5.3 Artifact-Type Variations

Only `project` artifacts support promote in v1.

| Type | Promote Allowed | Notes |
|------|----------------|-------|
| project | Yes | Only type with lifecycle |
| journal | No | No lifecycle semantics |
| snapshot | No | Immutable |
| restart | No | Immutable |

#### 3.5.4 Forbidden / Common Failure Patterns

| Pattern | Why It Fails | Error Code |
|---------|--------------|------------|
| Repeat promote (same transition) | Already promoted | LIFECYCLE_STATE_MISMATCH |
| Skip transition (seed to tree) | Invalid transition | LIFECYCLE_TRANSITION_NOT_ALLOWED |
| Promote snapshot | Wrong type | ACTION_NOT_ALLOWED |
| Missing transition field | Required | VALIDATION_ERROR |

## 4. Lifecycle and Governance Interactions

### 4.1 Actions That May Change Lifecycle

| Action | Can Change Lifecycle | Notes |
|--------|---------------------|-------|
| artifact.promote | Yes | Only action that mutates lifecycle_status post-creation |
| artifact.save | Initialize only (CREATE ONLY) | Sets lifecycle_status on create; does not support updates |
| artifact.update | No | Explicitly blocked per Mutability Registry |

### 4.2 Actions Explicitly Forbidden from Lifecycle Mutation

| Action | Enforcement | Governing Document |
|--------|-------------|-------------------|
| artifact.update | Gatekeeper blocks lifecycle_stage | Mutability Registry PROMOTE_ONLY |

### 4.3 Snapshot / Restart Implications

| Event | Snapshot Behavior | Restart Behavior |
|-------|-------------------|------------------|
| Lifecycle transition via promote | Event appended to qxb_artifact_event | N/A |
| Create | Immutable once written | Immutable once written |
| Update attempt | Rejected (CREATE_ONLY) | Rejected (CREATE_ONLY) |

## 7. Final Checklist (Pre-Execution)

### 7.1 All Actions

- [ ] `gw_action` is present and correct
- [ ] `gw_workspace_id` is valid UUID
- [ ] `artifact_type` is in allowed list for the action
- [ ] No server-owned fields are included (version, created_at, updated_at)
- [ ] `gw_user_id` is NOT included (actor derived from auth context)

### 7.2 artifact.save (CREATE ONLY)

- [ ] `artifact_id` is NOT present (forbidden; DB generates ID)
- [ ] `title` is present and non-empty
- [ ] `extension` is present with type-specific required fields
- [ ] For project: `extension.lifecycle_stage` is present
- [ ] If `parent_artifact_id` present, parent exists and is valid type
- [ ] NOT attempting to update an existing artifact (use artifact.update instead)

### 7.3 artifact.query

- [ ] `artifact_id` is present and valid UUID
- [ ] `artifact_type` matches actual type of artifact (or expect TYPE_MISMATCH)

### 7.4 artifact.update (ONLY update mechanism)

- [ ] `artifact_id` is present and valid (required; this is the only update path)
- [ ] `artifact_type` is "project" (only updatable type in v1)
- [ ] `extension` only contains allowed fields (operational_state, state_reason)
- [ ] `extension` does NOT contain lifecycle_stage (use promote instead)
- [ ] NOT using artifact.save for this operation (save is CREATE ONLY)

### 7.5 artifact.list

- [ ] `artifact_type` is present
- [ ] If paginating: reuse `as_of` from first page response
- [ ] `selector.limit` is <= 500

### 7.6 artifact.promote

- [ ] `artifact_id` is present and valid
- [ ] `artifact_type` is "project"
- [ ] `transition` is present and valid (seed_to_sapling, sapling_to_tree, tree_to_retired)
- [ ] Current lifecycle_status matches `from_state` of transition
